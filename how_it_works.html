Blockchain Basics

A blockchain consists of a series of blocks, each containing a record of transactions. Each block contains three essential components:

  • Data: This includes information specific to the blockchain’s purpose, such as transaction details.
  • Hash: A unique identifier for each block.
  • Previous Hash: This connects each block to the one before it, creating a chain. Any change to the data in one block will alter its hash, breaking the chain, which is why blockchain is known for its security and immutability [5].

How Transactions Are Added

  1. Verification: When a transaction is initiated, it is grouped into blocks and sent for verification to the nodes (computers) on the network. Nodes ensure that the transaction is valid before proceeding [6].
  2. Consensus: The nodes reach an agreement on the validity of transactions using consensus mechanisms. Common types include Proof of Work (PoW), where computers solve complex puzzles to validate a block, and Proof of Stake (PoS), where validators are chosen based on their 'stake' in the network [7].
  3. Addition to the Chain: Once consensus is achieved, the block is added to the chain, making it immutable and part of the permanent record [8].

Consensus Mechanisms

  • Proof of Work (PoW): This competitive process requires miners to solve complex mathematical puzzles. It is highly secure but energy-intensive. Bitcoin uses PoW as its consensus mechanism [9].
  • Proof of Stake (PoS): Validators are chosen based on the number of coins they hold and are willing to put up as collateral. This method is more energy-efficient than PoW and is used by blockchains like Ethereum 2.0 [10].

Proof Of Work (PoW) vs Proof of Stake (PoS)

The ability to mine a block is determined by the computational power of each miner.

The ability to mine is determined by how many tokens of this currency the user owns.

A reward is given to the miner who solves each block.

In PoS, the miner does not earn rewards but is paid with network fees.

In PoW, hackers need 51% of the network computing power to execute a 51% attack, which is highly unlikely but not impossible.

In PoS, a hacker would need to own 51% of all cryptocurrencies on the network to execute an attack, which is practically impossible.

“The blockchain does one thing: It replaces third-party trust with mathematical proof that something happened.”
